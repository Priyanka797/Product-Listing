# Product Listing Page

A responsive product listing page that fetches live product data from a public API and renders it dynamically using vanilla JavaScript.

## What it does

This page pulls a catalog of products from the [API NAME] API and builds the product grid on the fly — no hard-coded HTML for each item. Every card you see is generated by JavaScript from the live API response.

## What I built

- **Fetched live data from a REST API** using the JavaScript `fetch` method
- **Rendered content dynamically** by creating and inserting DOM elements from the API response
- **Styled a responsive product grid** that adapts to different screen sizes using CSS
- **Handled the full flow** from request → response → display, without any frameworks

## Built with

- **JavaScript (vanilla)** — API fetching and dynamic DOM rendering
- **CSS** — responsive grid layout and styling
- **HTML** — page structure

## What I learned

Working on this taught me how the front-end and a data source actually talk to each other — making an asynchronous request, waiting for the data, handling the response, and turning raw JSON into something a person can see and use. It's the difference between a static page and an interface that works with real, changing data.
